# Game1 Codex 迭代约束

这份文档抽取自旧自动化文档中仍然有效的部分，只保留对当前 `Game1` 有意义的原则。

## 1. 总原则

- 优先做算法级改进，而不是堆补丁或纯调参
- 每次改动都要有可运行验证
- 版本结论必须依赖足够样本，不能拿小样本乱判优劣
- 结论必须结合 replay，而不是只看 Elo 或胜率

## 2. 评测约束

- 两个 AI 的相对强度比较：至少 `100` 局才可下稳定结论
- 若要宣称“新版本明显更强”：
  - 对每个关键对手建议 `>=200` 局且胜率 `>55%`
  - 或在大样本 Elo 中稳定榜首
- 生产 Elo 与实验 Elo 不可横向直接比绝对值

## 3. 搜索类硬约束

若版本引入搜索、前瞻、beam、rollout、minimax 等：

- 单步 CPU 时间必须 `<=200ms`
- 必须有硬截止和保底回退
- 计时必须基于 CPU 时间口径，而不是只看 wall-clock

## 4. 文档同步要求

每轮有效迭代至少要同步：

- 设计假设
- 关键机制
- 评测证据
- replay 观察结论
- 下一步假设

## 5. 旧 AI 借鉴口径

当前只保留以下借鉴方向：

- `ANTWar-AI`
  - 固定槽位
  - 攻守状态切换
  - 留钱防 EMP
  - 建塔/升级/基地升级的统一决策视角

已删除 `Generals` 相关代码与规则文档后，不再要求继续从 `Generals` 文档中取材。
