# 版本清理报告（2026-03-04）

## 目标

- 减少无价值旧版本对评测与迭代的干扰；
- 保留“高强度主干 + 关键里程碑 + 当前候选”；
- 同步精简 `autolab/registry.json`，避免评测引用已删除目录。

## 价值判定标准

一个版本被判定为“有价值”，至少满足以下之一：

1. 生产评测长期处于上游（近期平均 rank 靠前，或历史/当前高 Elo）。
2. 具备独立算法贡献，且是后续有效分支的关键里程碑。
3. 属于当前活跃候选（本轮已验证有效，如 `v53`）。

### 保留价值细分（便于后续继续精简）

`A. 核心基线（不可删）`

- `v1`：规则适配最早完整基线，作为“退化检测锚点”必须保留。
- `v2`：当前长期高位锚点（最新全池评测 `eval_20260304_061650` 中 Elo 第一）。
- `v3/v4/v5/v6`：核心策略族（beam/counterfactual/2-ply）的主干演化链，后续分支大量依赖。

`B. 关键里程碑（建议保留）`

- `v8`：模式迟滞（hysteresis）分支起点，代表从“瞬时切换”到“状态机稳定化”。
- `v15/v17/v18/v19`：防守链路与 exchange/rebound/shell-floor 关键阶段；很多后续版本逻辑直接复用。
- `v21/v26/v29/v33/v40`：中期“硬化 + 反击车道”里程碑，属于当前策略族的结构节点。
- `v45/v47/v50`：后期 mutex/safeguard/piece-blend 节点，虽非最优，但保留用于消融对照。

`C. 当前候选（高优先保留）`

- `v53`：最新有效候选，头对头对 `v1` 两次 100 局分别 `64/100`、`63/100`；合计 `127/200`，胜率显著高于 50%。

注：

- 最新全池 Elo 与头对头结果存在一定不一致（评测池组成差异、样本配比不同），因此保留了“全局高位 + 对照价值 + 新候选”三类版本，而不是仅按单一 Elo 截断。

1. 由于历史版本已删除，回溯单点实验需依赖文档而非源码复跑。
2. 若后续需要“复活”某条分支，建议从文档记录中的关键里程碑版本重新分叉，而不是恢复全部旧版。

4. 文档中的历史日志（如 `round2_autolab_and_iterations.md`）仍会出现已删除版本名，这是保留历史可追溯性，不代表这些版本仍在评测池中。
