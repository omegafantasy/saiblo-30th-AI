# Game1 对旧 ANTWar-AI 的复刻覆盖度检查

## 1. 结论先行

结论：`cpp_v1_current` 只复刻了旧 `ANTWar-AI` 的“高层结构骨架”，还没有做到“尽可能完整复刻全部技巧与算法”。

更准确地说：

- 高层战略骨架：已复刻到位
- 旧搜索 / 危险预测 / 连续评估：没有完整复刻
- 超级武器细粒度博弈：只复刻了最关键的一部分
- 旧 AI 的组槽位约束、卖塔腾钱、应急暴风雪触发链：只复刻了部分思想，没有完整动作树

因此当前判断应该是：

- `cpp_v1` 是合理的 Game1 迁移基线
- 不是旧 `ANTWar-AI` 的完整复刻版

## 2. 已复刻到位的部分

### 2.1 固定槽位体系

已复刻。

证据：

- `Game1/Ant-Game/SDK/utils/constants.py` 中的 `STRATEGIC_BUILD_ORDER`
- `Game1/Ant-Game/AI/ai_cpp_v1.py` 会把这些槽位送给 C++
- `Game1/antgame_ai_cpp/v1/ai_v1.cpp` 明确依赖 `slots` 做建塔 / 升级

### 2.2 攻守状态切换

已复刻，但比旧版更轻。

旧版：

- 基地血量领先/落后
- 击杀/存活兵力差
- 晚期回合特判
- `attack_flag` 滞回

当前 `cpp_v1`：

- 基地血量差
- 前压 / 敌压比较
- 晚期回合切换
- `attack_commit_` 滞回

判断：核心思想已在，但状态机比旧版简单很多。

### 2.3 留钱防 EMP

已复刻。

证据：

- 旧 AI 核心是 `safe_coin()` / `reserved`
- 当前 `cpp_v1` 直接使用 SDK 的 `safe_coin_threshold(player)`
- 候选评估里对低于 reserve 的操作有明显惩罚

### 2.4 建塔 / 升级 / 基地升级统一决策

已复刻。

证据：

- `cpp_v1` 会把 build / upgrade / combo / base-upgrade / superweapon 一起放进候选比较
- 这是旧 AI 最值得保留的结构之一

## 3. 只复刻了一部分的部分

### 3.1 超级武器

只复刻了一部分。

旧 AI：

- 闪电风暴
- EMP
- Deflector
- EmergencyEvasion
- 还带有应急卖塔、时机门控和终盘特殊触发

当前 `cpp_v1`：

- 重点只放在 `LightningStorm` 和 `EMP`
- `Deflector/Evasion` 没有进入主决策
- 没有完整复刻旧版的应急链条

### 3.2 组槽位 / 家族互斥

只复刻了顺序，没有完整复刻组约束。

旧 AI 里 `get_group()` 明确把一组点位当成互斥/耦合对象处理。

当前 `cpp_v1`：

- 只有顺序和分支，没有真正的组级动作约束
- 因此“同一路线过密/缺少横向分布”这类旧 AI 会避免的问题，当前版本仍可能出现

### 3.3 危险预测

只保留了启发式，没有保留旧模拟链。

旧 AI：

- `Simulator`
- `Node::evaluate`
- `select_expand`
- `fail_round / ruin_round / dis_vals` 这类前瞻性危险评估

当前 `cpp_v1`：

- 只有静态候选打分
- 没有中短期 fail-round 预测

## 4. 基本没有复刻的部分

### 4.1 旧搜索框架

没有复刻。

这是当前与旧强 AI 最大的差距。

旧 AI 的强点之一并不是参数，而是：

- 先扩展根动作
- 再做选择性展开
- 在极小时间预算里优先看危险节点

当前 `cpp_v1` 完全没有这一层。

### 4.2 卖塔腾钱与应急连招

没有系统复刻。

旧 AI 在某些超级武器、基地升级、应急防守场景下，会主动卖塔换资源。

当前 `cpp_v1`：

- 没有 downgrade/sell 相关动作链
- 因此在“资金差一点但必须应急”的场景下，会少很多旧版能做出来的动作

## 5. 为什么这次没有继续强推 `v2`

原因很直接：

- 我补做了一版 `cpp_v2_antwar_structured`
- 方向是把组槽位、攻守切换、防御超武这些结构技巧补进去
- 但本地冒烟已经出现对 `example` 的退化

结论：

- 这个方向并非没价值
- 但当前实现还不稳，不能直接当“更完整复刻成功”

## 6. 当前正确结论

如果问题是“现在是否已经尽可能复刻旧 ANTWar-AI 的所有技巧、思路、算法”，答案是：没有。

如果问题是“现在是否已经有一个以旧 ANTWar-AI 为主线、适配新规则并可继续迭代的可信基线”，答案是：有，就是 `cpp_v1_current`。

## 7. 下一步最值得补的三项

1. 轻量 fail-round / danger 预测，而不是完整回退到旧模拟器
2. 组槽位互斥与分布约束
3. `Deflector/Evasion` 与应急卖塔链
