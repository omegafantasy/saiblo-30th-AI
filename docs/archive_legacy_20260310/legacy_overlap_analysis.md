# 历史 AI 与蚁洋陷役2重合度分析

## 1. 分析范围

- 历史项目：
  - `/www/past_AIs/ANTWar-AI`
  - `/www/past_AIs/Generals-AI`
- 目标项目：
  - `/www/Ant-Game`
  - `/www/docs/蚁洋陷役2 - Saiblo.mhtml`

## 2. 规则层重合度

### 2.1 与 ANTWar 的重合

高重合点：

- 文案叙事与术语（工蚁、超级武器、塔防风格背景）。
- Saiblo 页面对“Antwar2”的命名与历史塔防表述连续。

低重合点：

- 当前代码没有塔类型、索敌、AOE、工蚁自动寻路核心逻辑。
- 当前代码是 15x15 四方向占格与将军系统。

结论：

- `ANTWar-AI` 的“策略目标函数”和“塔防微操逻辑”可借鉴很少。
- 更适合作为“线上数据接口/评测流程经验”参考，而非决策内核。

### 2.2 与 Generals 的重合

高重合点：

- 地图规模与四方向移动。
- 将军/副将/油井(农夫)的占格与经济框架。
- 操作码体系（移动、升级、技能、科技、超武、征召）。
- 平局裁决优先级：兵力 > 占格 > 资源。

结论：

- `Generals-AI` 与当前目标在“可执行规则”上高度同构。
- 初版 C++ AI 迁移应优先从 Generals 的协议解析与局面评估思路出发。

## 3. 工程资产重合度

### 3.1 可直接迁移

- `Generals-AI/include/json.hpp`
  - 可作为 C++ 侧 JSON 解析头文件。
- `Generals-AI` 的操作码建模思想
  - 与 `Ant-Game/logic/ai2logic.py` 几乎一致。
- 评估函数骨架
  - 兵力优势、占格扩张、将军生存优先级可复用。

### 3.2 需要改造后迁移

- `zdata.py`（旧 Saiblo 数据脚本）
  - 已迁移为根目录 `saiblo_tools.py` 的 token 回退源（参数/环境变量优先，`zdata.py` 兜底）。
  - 原脚本内硬编码字段仍建议仅作为本地私有凭据使用。
- `zlocal.py`（旧本地评测）
  - 当前内容针对 Pacman，不可直接复用；但其“并行跑多局+汇总”流程可迁移。

### 3.3 不建议迁移

- `ANTWar-AI/main.cpp` 的塔位编码与专用规则分支。
- 任何依赖六边形地图/防御塔行为的硬逻辑。

## 4. 对初版 AI 的策略建议

第一版应追求“稳定拿分”而非复杂花招：

- 主将附近快速扩张，优先吃空地与薄弱敌格。
- 保持主将格安全（至少留 1 兵），避免送主将。
- 有金币时在己方关键格征召副将，提升中后期增长。
- 以兵力净收益为核心，优先可确定占领的移动。

## 5. 迁移路线（已落地）

- 协议侧：新建 C++ bot，适配 `seat seed` + JSON 状态 + 4 字节长度前缀输出。
- 本地评测侧：用 Python 桥接 C++ 进程接入 `run_match` 生态，保留现有批评测工具。
- Saiblo 侧：新脚本使用环境变量 Token，支持拉取对局、创建房间并开赛。

## 6. 结论

- 目标游戏在“文案叙事”上更像 ANTWar，在“代码可执行规则”上更像 Generals。
- 因此，策略代码优先继承 Generals 系列思想；流程工具与线上经验再吸收 ANTWar 项目的部分方法。
