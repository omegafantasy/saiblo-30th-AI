# Generals-AI 深度分析与 v64 迁移说明

## 1. 分析范围

- 旧 AI 源码：`/www/past_AIs/Generals-AI/main.cpp`（约 4.8k 行）
- 当前规则实现：`/www/Ant-Game/logic/*.py`
- 新迁移版本：`/www/ai_cpp/v64/ai_v64.cpp`

目标：不是“参数复刻”，而是复刻旧 Generals-AI 的决策结构与战术优先级。

## 2. 旧 Generals-AI 的核心决策结构（真正强点）

### 2.1 顶层决策流水线（强优先级）

`simple_ai()` 的主链路为：

1. `use_weapon`
2. `new_kill`（直接斩首/斩首链）
3. `search`（短视野模拟）
4. `tp`（逃生/重定位）
5. `use_tech_main`
6. `upgrade`
7. `action`
8. `post_escape`
9. `upgrade`（二次）

这个顺序的意义：
- 先判“有没有立即终结局面”的战术机会；
- 再做短期博弈与保命；
- 最后做经济与常规扩张。

### 2.2 威胁评估不是简单势力和，而是“可达兵力打包”

`compute_threat_val()` 的核心不是邻域求和，而是：

- 先按距离收集敌方潜在进攻来源；
- 融合技能/金币可支付性（统率、弱化、击破、征召副将等）；
- 用“容量=敌方机动步数”的背包式 DP 估计单回合最大威胁。

这比单纯 `sum(army/distance)` 更贴近实战威胁上限。

### 2.3 斩首模块是“组合动作搜索”

`new_kill()` 会调用：

- `best_dfs()`：多步兵力移动链
- `best_skill_dfs()`：技能与行动组合

本质是“能否在本回合拼出可执行斩首序列”。这也是旧 AI 爆发力的关键来源。

### 2.4 `search()` 是“我方升级分支 × 敌方应对分支”的短期模拟

不是标准 minimax，但做了：

- 若干上层决策候选（我方升级/策略开关）
- 若干下层应对候选（敌方升级/应对）
- `sim_operation` 进行多回合短模拟
- 综合局面差值、威胁状态、收益与风险后择优

### 2.5 经济与行动逻辑是“安全门控”的

`upgrade()`、`action()` 里大量使用：

- 主将战斗值与安全阈值
- 分将优先级排序
- 前线/探索/支援动态切换

即：经济行为依赖“当前是否安全”，不是固定回合表。

## 3. 当前游戏与 Generals 的关键差异（影响迁移）

### 3.1 高重合（可直接迁移）

- 15x15 格子地图
- 主将/副将/农夫结构
- 四方向军队移动
- 升级、科技、战法、超武、征召副将

### 3.2 需要特别处理的差异

1. 规则文案混入旧 AntWar 塔防叙事，真实执行以 `Ant-Game/logic/` 为准。
2. 技能细节：`SURPRISE_ATTACK` 不能突到有将军的格子（不能直接冲主将格）。
3. 当前 C++ AI 通常只模拟“军队一步移动”，未完整模拟“技能链 + 将军移动链”。
4. 线上/本地可能有经济节奏差异风险（历史文档提到过双加金币路径）。

## 4. v64 的迁移策略（本次落地）

v64 基于 v63 主干保留：

- overlay 候选池 + 一步敌我应答评估
- 线程 CPU 时钟硬截止（200ms）

并新增 3 个“Generals 结构化模块”

### 4.1 `kill-sequence beam`（对应旧 `new_kill` 思想）

新增 `search_forced_main_kill_sequence()`：

- 在当前回合步数预算内做小束宽搜索；
- 优先探索“贴主将攻击”与高压候选；
- 找到可执行斩首序列则直接输出。

这是从“单步最优”向“本回合组合终结”迁移的关键。

### 4.2 战法层（对应旧 `use_tech_main` / 战术层）

新增技能决策：

- `ROUT` 对敌主将格降兵（开路）
- `COMMAND` 攻击爆发前置
- `DEFENCE` 主将高压防守
- `WEAKEN` 高压时压制敌方进攻圈

并在状态里维护技能持续/冷却近似，用于后续评估。

### 4.3 主将重定位（对应旧 `tp/action` 的机动意图）

新增 `try_move_main_general()`：

- 基于威胁图 + 前线度 + 可达路径，决定主将换位；
- 高压时偏保命，平稳时偏前压。

## 5. 与旧 Generals-AI 的差距（仍未完全复刻）

1. 未完整迁移 `best_skill_dfs` 级别的技能组合树。
2. 未做完整“超武释放价值搜索”。
3. 仍是轻量短视模拟，不是完整 `search()` 多回合分支框架。

因此 v64 目前是“结构复刻版（第一版）”，不是“行为逐行等价复刻”。

## 6. 本轮实测结果（2026-03-04 UTC）

评测范围：`runtime_scope=v64_probe`

- `eval_20260304_105419`（v64 vs v1/v53）
  - vs `cpp_v1_current`: 60 局，`50-10`，胜率 `83.33%`
  - vs `cpp_v53_overlay_countercheck`: 60 局，`33-27`，胜率 `55.00%`
- `eval_20260304_105518`（v64 vs v63）
  - vs `cpp_v63_overlay_presort_earlystop`: 60 局，`43-17`，胜率 `71.67%`

说明：
- 在“最新 overlay 系列”与 `v1` 上，v64 都出现了正向突破信号；
- 但样本仍属于中等规模，是否作为生产 champion 仍应按你既定大样本门槛继续复验。

## 7. 后续复刻迭代（v65-v69）与结论

### v65：Generals `search` 风格宏分支模拟

- 设计：把“技能/主将位移/征召”做成候选分支，并用短 rollout 打分选择。
- 结果：`eval_20260304_111043`，对 `v64/v1/v53` 全部负胜率（尤其对 `v64` 仅 `22%`）。
- 诊断：宏分支模拟带来的“决策预算消耗 + 误判”超过收益，属于失败复刻。

### v66：补 `use_weapon` + `upgrade` 子模块

- 设计：引入核弹决策与安全副将产能升级（对应旧 `use_weapon/upgrade`）。
- 结果：`eval_20260304_111429`
  - vs `v1`: `82%`
  - vs `v53`: `60%`
  - vs `v64`: `42%`
- 结论：对旧强度族群有提升，但关键对位 `v64` 退化，不能主线替代。

### v67：保守门控版（抑制误触发）

- 设计：核弹与副将升级都加高风险门控，减少错误触发。
- 结果：`eval_20260304_111735`
  - vs `v64`: `25/60`（`41.7%`）
  - vs `v66`: `29/60`（`48.3%`）
  - vs `v1`: `53/60`（`88.3%`）
  - vs `v53`: `36/60`（`60.0%`）
- 结论：仍未修复对 `v64` 的负回归。

### v68 / v69：极保守 fallback 试验

- 设计：进一步收紧触发条件（后期且优势局）或仅保留严格核弹策略。
- 结果：
  - `eval_20260304_112006`（v68 vs v64）：`38/80`（`47.5%`）
  - `eval_20260304_112203`（v69 vs v64）：`38/80`（`47.5%`）
- 结论：仍略弱于 `v64`，说明当前“武器/经济附加层”尚不足以击穿 `v64` 主干。

## 8. 当前最关键的复刻缺口

从实测看，真正缺口不是“是否会放超武或做经济升级”，而是：

1. 旧 Generals `best_skill_dfs` 的“技能+移动联合斩首”没有完整复刻；
2. 旧 `search()` 的对抗式分支模拟在我们当前轻量实现上误差较大；
3. 目前版本在强对手对位上，仍更依赖 v64 的基础局部博弈能力。

因此下一阶段应优先复刻“局部战术组合搜索（特别是技能链）”，而不是继续扩展全局经济/超武规则层。
