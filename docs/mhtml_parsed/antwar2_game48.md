# 蚁洋陷役2

[游戏代码仓库](https://github.com/LorenzLorentz/Ant-Game)

### 游戏背景

在微观的自然世界中，你将化身为运筹帷幄的蚁群指挥官，展开一场名为“AntWar”的硬核塔防博弈。这不仅仅是简单的攻防，更是对动态策略与资源调配的极致考验。你的核心部队——“工蚁”机器人，并不由你直接操纵，而是依赖基于真实自然界信息素的寻路算法，在复杂的环境中自主摸索、适应并向敌方基地发起冲锋。面对不断自适应的敌方“工蚁”，你需要通过升级基地、部署防线、释放超级武器，在进攻与防御的资金投入中找到完美平衡，最终摧毁敌方主将，赢下这场微观世界的史诗战争。

1. 地形与环境的多样性： 棋盘由不同地形构成（如平原、沼泽、山地），每一局的战场都充满变数。你需要充分利用地形优势，例如注意沼泽对军队带来的负面惩罚。

2. 基地升级与科技树： 通过投入资金升级主将与基地，你可以优化生产流水线，大幅提升“工蚁”机器人的组装产出速度；也可以列装高级护甲，直接提升工蚁的生命值上限。

3. 动态防御与塔防建设： 为了抵御敌方工蚁的无情冲锋，你可以建造多种防御塔。无论是“一炮一个”的重型加农炮、专注射速的轻机枪，还是擅长范围打击的经典迫击炮，都任你差遣。由于新建防御塔的价格呈指数级增长，你必须审慎规划建造点位。

4. 超级武器定乾坤： 当战局陷入胶着，你可以动用造价不菲、冷却漫长的超级武器。无论是造成高额范围伤害的“闪电风暴”，还是能瞬间瘫痪敌方防御塔的“EMP轰炸”，只要在关键时刻释放，便能一击制胜。

### 核心玩法

游戏的棋盘大小为 `row x col` 格局。双方玩家各拥有一名主将 作为己方基地的核心，盘面中还会出现副将与农夫。 本游戏采用回合制进行，先手与后手玩家交替下达指令。你的目标是摧毁敌方主将，同时保护己方主将免遭敌方工蚁的啃食。

### 基本规则

在属于你的玩家回合内，你可以自由进行以下操作来确立优势：

- 军队与将领调度： 指挥军队移动，或进行将军的移动与升级（包括提升产量、防御和机动性）。
- 战术博弈： 释放将军战法、研发核心科技，或征召新的副将加入战场。
- 终极打击： 使用超级武器改变战场局势。

回合结算机制： 当双方都完成一回合的操作后，游戏逻辑将进行统一结算：

- 兵力变动： 结算本轮的产兵增益、沼泽地形惩罚，以及每轮或每10轮的自然兵力增长。
- 状态更新： 刷新所有技能与超级武器的持续时间和冷却时间。
- 重置与记录： 重置双方的移动步数，并生成本轮的对局回放记录。

### 结算流程

在《AntWar》的微观战场上，每一回合的交锋都受到严密的系统逻辑监控。回合结算不仅关乎战术指令的落地，更决定了战局的生死走向。整个流程分为“玩家操作”与“回合结算”两个核心阶段：

玩家操作阶段

- 指令等待与高压校验： 系统首先等待“玩家 0”下达操作指令。战场容不得半点失误——若玩家程序出现崩溃、运行超时、返回不符合协议的数据，或企图执行非法操作，系统将毫不留情地直接判负。
- 战术指令执行： 通过校验后，系统将忠实执行“玩家 0”的部署，包括建造或升级防御塔、升级基地、释放超级武器等一系列操作。
- 对手轮替交锋： “玩家 0”的操作落地后，系统将对“玩家 1”执行完全相同的等待、校验与执行步骤。

回合结算阶段

- 气象与环境打击： 双方操作完毕后，系统首先结算“闪电风暴”等全局超级武器造成的毁灭性效果。
- 防线火力交织： 严格按照防御塔的建造时间顺序，依次结算所有防御塔的攻击伤害。
- 战损与战利品： 若本回合火力网造成敌方“工蚁”死亡（生命值降至非正数），防守方将根据该阵亡工蚁的等级，立即获得相应的金币赏金。
- 自然老化机制： 系统会检视所有工蚁的寿命。如果当前回合数减去该工蚁的生成回合数大于 32（即超过最大年龄），它将被无情地标记为“老死”。
- 大军压境与异常状态： 按照工蚁的生成顺序依次结算移动。已死亡的工蚁将停止步伐；被冻结的工蚁本回合将被禁足，但在结算完毕后会自动解除冻结状态。
- 兵临城下与斩首： 若有幸存的工蚁成功突入敌方基地，将立刻削减敌方基地血量。一旦基地血量归零，游戏即刻宣告终结！
- 信息素挥发与战场清理： 结算全场信息素的分布与浓度变化，并从盘面上彻底移除已死亡或已达阵的工蚁。成功达阵的工蚁将为己方结算一笔丰厚的金币收益。
- 兵源流水线： 根据双方当前的基地等级，系统会尝试自动生成并投放新一批的工蚁机器人。
- 回合资源红利： 激战之余，双方玩家均会自动获得 1 枚金币的战备基础收益。
- 时间推进与终局裁决： 结算末尾，回合数加 1。当回合数达到 512 的极限阈值时，系统将强制介入，执行最终的胜负裁决并结束整场游戏。

### 胜负判定规则

- 主将被毁： 若任意一方的“主将”被消灭，则存活的另一方立即获得游戏胜利。
- 回合上限限制： 游戏在 500 回合内未分出胜负时，将强制结束并采用平局裁决机制。
- 平局裁决优先级： 当触发 500 回合平局判定时，系统将严格按照 总兵力 > 占领格数 > 金币数量 的优先级来决定最终胜者。若所有数据完全一致，则默认判先手玩家获胜。

## 开发者与 AI 接入指南

为了方便开发者编写 AI 并进行强化学习（RL）训练，我们提供了完整的 SDK 环境与评测工具。

### 1. 数据与核心模块架构

- 核心数据结构 (`logic/gamedata.py`)： 包含 `Cell`、`Generals` (Main/Sub/Farmer)、`SuperWeapon` 以及各种枚举类型（方向、地形、技能、科技、武器）。
- 游戏状态维护 (`logic/gamestate.py`)： 维护棋盘、将军列表、金币、科技、武器状态、剩余移动步数以及回放文件路径等全局信息。
- 指令执行入口 (`logic/ai2logic.py`)： 统一通过 `execute_single_command` 接收参数。涵盖移动（含 BFS 合法性检查）、升级、战法释放、科技研发、超级武器使用及征召副将。
- 战斗计算与回合更新 (`logic/computation.py` & `logic/gamestate.py`)：`update_round(state)` 负责每轮的各项数值衰减与重置，并通过 `logic/generate_round_replay.py` 追加记录 jsonl 格式的回放文件，便于后续重播分析。
- 本地对战运行器 (`logic/runner.py`)：`run_match` 负责初始化对局、交替调用双方 AI 产生命令、执行结算，并将回放保存在 `replays/` 目录下。

### 2. SDK 强化学习环境

本游戏基于 PettingZoo 构建了环境：

- 代理与奖励： 代理命名为 `player_0` 和 `player_1`。终局奖励：胜者 +1，败者 -1，平局 0（执行非法动作 -0.01）。
- 动作编码 (长度 8)： 索引 0 代表操作类型 `type (0..8)`。例如：0 代表 EndTurn，1 代表 ArmyMove，6 代表 SuperWeapon 等。
- 观测空间： 包含盘面所有权、地形、军队数量、金币、剩余步数等全方位信息。

依赖安装：`pip install pettingzoo gymnasium torch`

最小使用示例：

```python
from SDK import env
e = env(render_mode="ansi")
e.reset()
for _ in range(20):
    agent = e.agent_selection
    a = e.action_space(agent).sample()
    a[0] = 0  # 偏置为 EndTurn 以推进游戏
    e.step(a)
    if any(e.terminations.values()) or any(e.truncations.values()):
        break
print(e.render())
```
