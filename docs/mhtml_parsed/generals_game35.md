# 游戏文档

## 一、游戏背景

特瓦提大陆是一个富饶的幻想世界，拥有丰富的历史和文化底蕴。这片土地曾经是古代文明的交汇处，留下了许多古老的城市废墟和神秘的遗迹。其地理位置及其丰富的石油资源使其成为地缘政治的关键点，备受世界瞩目。而这导致了周边一些势力对该国家虎视眈眈，为了防备外部势力和特瓦提本土势力之间的纷争，大陆决定进行一次军事演习。这次演习不是单纯的练兵，更是对各位指挥官资源统筹和带兵能力的考验。 在这场回合制的军事演习中，每一个指挥官的决策和行动都将产生深远的影响。这个国家的命运就如同这场演习一般，悬而未决，等待着智者的谋略和英雄的崛起。

- 地形多样性：特瓦提的地形多种多样，从富饶的平原到泥泞的沼泽，再到恐怖的流沙。演习中，指挥官们必须灵活运用地形，制定巧妙的战术，以确保胜利。
- 资源战略：某些地区被发现拥有丰富的石油资源。石油是工业的血液，也是陆军装备升级、空军执行任务的必需品。指挥官们需要争夺这些油田，以保证己方充足的燃料供应，而石油的消耗则成为一项关键的资源管理考量。
- 兵力与将领：指挥官可以将士兵部署在各个地区，而有些地区则隐藏着具有指挥才能的将领。指挥官们需要谨慎选择攻击目标，同时可以试图招揽中立将领，扩大自己的指挥团队。
- 战术能力：为了在战局中取得优势，指挥官们可以利用积累的石油资源发动各种战术能力，比如空袭轰炸敌军集结区，破坏敌方指挥系统，或者进行侦察获取对手的情报。 整个演习的目标是通过占领关键区域，俘虏敌方总指挥官来获得胜利。然而，演习中各种变数不断，指挥官们需要灵活应对地形的挑战、战术的调整以及资源的管理，以确保最终的胜利。这场虚构的战争演习将成为一场智谋与战术的较量，决定着特瓦提未来的安危。 本场演习采用回合制进行，双方玩家分为先手和后手，先手玩家先进行其全部操作传递给游戏逻辑，游戏逻辑结算后，后手玩家再进行操作，并进行结算。两个玩家都操作后，一个回合结束。游戏逻辑将会根据将军和油田数量对地图进行更新，并进行下一回合的游戏，

## 二、游戏地图简介

演习地图由15*15小陆地方格组成，每一块陆地的类型可分为平地，沼泽，流沙。不同的地形有着不同的特性 。

- 每10回合，一方所有占领的方格的兵力都会默认增加1。
- 每回合，处于流沙地上的兵力都会默认减少1，在升级科技后，这样的debuff可以消除。
- 默认情况下，沼泽不能停留军队，因此将军也不可以被移动到沼泽上，但是这一点可以通过升级科技改变。
- 初始时，地图上有十个油田（7个在平地上，3个在沼泽中），四个中立副将，以及双方各一名主将。油田随机拥有3-5名中立军队，副将随机拥有10-20名中立军队。

## 三、移动规则

- 军队移动规则：默认情况下，每回合每个玩家可以移动两步，如果升级对应科技，可以移动的步数也会增加。玩家可以命令若干（全部或部分）军队从已经占领的方格向其相邻方格（上下左右）进军。当军队移动时，保持军队总数不变的原则，但平地需要至少留下一个兵镇守。（一个兵的时候无法再移动）
- 将军移动规则：玩家也可以命令将军进行移动，将军的移动与军队的移动是独立的，但是己方的将军只能移动到己方占领的格子上，不可以移动到中立的或敌方占领的格子上。

## 五、经济系统

本游戏中，主要的资源是石油。获取石油的方式只有占领地图里中立的油田。默认等级的油田每回合生产1单位石油，当然，你也可以通过升级该油田，使得每回合产出更多的资源。你可以使用这些资源升级科技或将军，也可以用资源来使用技能，获得作战中的优势。

### 1. 升级将军或油田。

- 油田：默认情况下生产1单位石油/回合，防御力为1

  - 产量升级

    - T1：花费10单位石油，生产能力提升至2单位石油/回合
    - T2：花费25单位石油对T1进行升级，生产能力提升至4单位石油/回合
    - T3：花费35单位石油对T2进行升级，生产能力提升至6单位石油/回合

  - 防御升级

    - T1：花费10单位石油，油井的防御力增加为1.5
    - T2：花费15单位石油对T1进行升级，油井的防御力增加为2
    - T3：花费30单位石油对T2进行升级，油井的防御力增加为3

- 副将：将军默认情况下每回合征召兵力1个，防御力为1

  - 攻击范围：默认为以将军为中心5*5的格子
  - 升级

    - 可以使用50单位石油在平地上征召一位新的将领
    - 产量升级

      - T1：花费40单位石油，该将军1回合出兵2个
      - T2：在T1基础上，花费80单位石油，该将军1回合出兵4个

    - 防御升级

      - T1：花费40单位石油，使得该将军防御力增加为2
      - T2：在T1基础上，花费100单位石油，使得该将军防御力增加为3

    - 行动力升级

      - T1：花费20单位石油，将该将军每次行动格数上限增加为2
      - T2：再花费40单位石油，将该将军每次行动格数上限增加为4

- 主将：升级方式和战法使用与副将相同，但是升级使用的单位石油数目减半（战法使用单位石油相同）。

### 2. 使用战法：

- 在比赛过程中，可以消耗一定数量的石油，将军可以通过使用这些资源产生特殊的效果，合理地移动将军和使用战法，能够起到决定战场局势的效果。每个将军独立使用战法，冷却也独立计算，但都需要消耗一定数量的资源。

  - 突袭：使用20单位石油作为燃料，将军可带领本格兵力快速移动到攻击范围内的任意可到达位置（可到达位置：未被将军占领的可到达格子，且将军带领的兵力能够将该格子占领），冷却5回合。
  - 击破：使用15单位石油，将军派遣空袭部队对攻击范围内任意一个格子进行空袭，使得目标格子兵力-20，冷却10回合
  - 统率：使用30单位石油，使得10回合内，将军射程内所有我方兵种攻击力*1.5，冷却10回合
  - 坚守：使用30单位石油，使得10回合内，将军射程内所有我方兵种防御力*1.5，冷却10回合
  - 弱化：使用30单位石油，使得10回合内，将军射程内所有敌方兵种攻击力，防御力*0.75，冷却10回合

### 3.科技升级：

- 军队行动力升级

  - T1：花费80单位石油，使每回合可以移动3步。
  - T2：花费150单位石油，使每回合可以移动5步。

- 免疫沼泽：花费100单位石油，使得军队和将军可以占领和通过沼泽
- 免疫流沙：花费75单位石油，使得在流沙上的军队不再持续减少
- 解锁超级武器：花费250单位石油，获得超级武器的使用权，50回合冷却（初次使用需要冷却10回合），就绪后可以在己方回合开始时使用。

  - 核弹：摧毁3*3范围内的所有兵和副将，油井，对主将伤害减半，并留下5回合辐射，辐射会使每格每回合-3兵力（会使被摧毁的格子变为中立）
  - 攻击强化：5回合内，使3*3范围内的所有兵种得到强化，攻击力和防御力变为原来的3倍，效果结束时还原。（注意：攻击强化的效果是对格子而言的）
  - 超时空传送：将某格军队传送到除将军占领处外的任何位置 ，并无视敌方兵力占领此格，但被传送的部队会眩晕2回合。
  - 时间停止：使得3*3范围内双方军队10回合内不能行动，将军不能移动，使用战法。

## 六、胜负判定

AI限制运行时间1s，空间512MB，超过限制直接判负。

如果AI返回非法操作直接判负。

击败对方主将即算胜利。为避免游戏时间过长，在比赛进行500回合后比较双方兵力和占领的格数，兵力多的获胜，若兵力一样则占领的格数多的获胜，若格数一样，则石油更多的获胜。若石油相同则总用时短的获胜，若全部相同则先手方获胜。

## 八、通讯协议及游戏流程

### 通信流程

- 逻辑给双方发送初始化信息并等待先手方玩家发送操作
- 逻辑接收到一方操作后进行结算
- 结算后如游戏继续，则将刚刚收到的操作转发给另一方，并开始计时等待接受另一方操作
- 如果中途游戏结束，则对局直接终止

### 初始化信息及回放文件格式

```json
{
    Round: int,
    Player: int,
    Action: [code, param1, param2, ...],
    Cells: [
            [[x, y],Player(-1/0/1), Soldier_num]
            …
            ],
    Generals: [
                {
                    Id: int,
                    Player: int,
                    Type: int (123/主副油),
                    Position: [x, y],
                    Level: [int, int, int(油田最后一个数为1)],
                    Skill_cd: [int, int, int, int, int]（油田全是0）,
                    Skill_rest: [int, int, int](统率、坚守、弱化的剩余回合)
                    Alive: bool,
                 }
                 …                
               ],
    Weapons: [
                {
                    Type: int,
                    Player：int，
                    Position: [x, y], 
                    Rest:int
                }
                …               
             ]
    Weapon_cds:[int, int],
    Tech_level:[[int（行动力）, int（免疫沼泽）, int（免疫流沙）, int（解锁超级武器）], [int, int, int, int]]        
    Coins: [player1, player2],
}
```

- 初始化时

  - Round为0
  - Player为先后手信息，先手为0，后手为1
  - Action置为[8]
  - Cells为所有格子
  - Generals为所有将军，油田
  - Weapons为空
  - 其余值为字面意思

- 回放文件中

  - Round为对应回合数
  - Player为进行该操作的玩家，系统结算操作的player为-1
  - Action为具体操作，格式为[操作数，操作码1，操作码2，...]
  - Cells为当次操作中兵力或所属玩家发生变化的格子
  - 其余与初始化时相同

### 逻辑转发和AI发送操作格式

- 共n行代表n个操作
- 每行为空格隔开的多个整数，值与前面表格对应
- 无论发送还是接收，最后一行一定为操作8
- AI一回合只能发送一次操作，请一次性发送当回合所有操作

### 关于随机生成地图

- 目前的地图采用随机生成+后期处理的模式生成
- 先纯随机生成40%的沼泽地形和15%的流沙地形
- 第一遍遍历所有格子，对于所有沼泽，若周围八个格子中沼泽数目<3，则置为平地
- 第二遍遍历所有格子，对于所有平地和流沙，若周围八个格子中平地，流沙数目<4，则置为沼泽
- 最后生成两方主将，纯随机位置，并将随机到的位置置为平地，同时保证两方将军的曼哈顿距离大于18
- 若主将之间不经过平地连通，则随机打通一条道路为平地
- 油田在平地上随机生成7个，在所有沼泽中随机生成3个，将军在平地上随机生成4个
