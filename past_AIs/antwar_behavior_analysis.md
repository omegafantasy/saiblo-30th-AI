## 11. Key Observations

### Methodology Limitations

- **Super weapon actions not captured in stderr**: The `try_attack`/`try_use_superweapon` functions return operations directly, bypassing the tree search. Their actions are not printed via `action:` lines in stderr. The super weapon usage table is empty for this reason. Emergency storm IS detected because it has its own stderr line.
- **Base upgrades not captured**: Ant level upgrades and gen speed upgrades go through `try_attack` which returns before the tree search runs. These are also not in `action:` lines.
- **HP differential always 0**: In self-play with identical AI, the average HP difference is symmetric (0.0 at every sampled point). Individual matches DO have HP differences but they average out.

### Strategic Insights from 200-Match Self-Play

**1. Spatial Strategy: Mid-Field Dominance**
- Top positions: ML1(393), MR1(379), ML2(305), MR2(305), M3(298), M2(289)
- The AI strongly prefers mid-field positions (ML, MR, M groups) over near-base (C, L, R) or far-forward (FL, FR, F) positions
- This aligns with code analysis: the evaluation function rewards distance from base (+0.4/distance) and penalizes clustering (proximity penalties)
- Forward positions (FL, FR) become more common in later rounds (300+), suggesting progressive expansion

**2. Tower Type: Mortar Meta**
- Mortar upgrades dominate (1969 upgrades, 56% of all upgrades) over Quick (1096, 31%) and Heavy (355, 10%)
- Level 3 upgrades are rare (Sniper:46, Missile:21, MortarPlus:7, etc.) — the AI keeps towers at level 2
- This is economically rational: level 3 costs 200 coins (vs 60 for level 2), and the evaluation penalizes level 3 towers at -260*0.15 = -39 vs level 2 at -60*0.15 = -9
- Mortar preference makes sense: 16 ATK + AOE + 3 range is efficient for clearing ant waves on the hex grid

**3. Economy Phases**
- **Opening (R0-R30)**: Coins drop to ~30 (initial tower builds), then recover
- **Mid game (R30-R120)**: Coin oscillations around 30-80 (building/upgrading/destroying cycles)
- **Late game (R120-R450)**: Steady accumulation to ~273 coins (towers established, income exceeds spending)
- **Endgame (R450-R510)**: Sharp coin drop to ~130 (attack mode triggers at avg R411, spending on ant upgrades + super weapons)

**4. Opening Repertoire**
- **Most common first move**: Build at MR1 (right mid, (6,14) for P0) or ML1 (left mid, (6,4) for P0)
- **Second move**: Build at M2 or M3 (central mid-field) around rounds 8-10
- First tower at mid-field (not near base) indicates the AI's evaluation function correctly identifies that forward positioning is more valuable than defensive closeness

**5. Game Length and Symmetry**
- All 200 matches go to round 512 (max), confirming tight defensive play in self-play
- P1 has slight advantage (48.5% vs 38.5%) — the second-mover advantage from seeing P0's actions before deciding
- 26 ties (13%) indicate many matches are extremely close

**6. Action Intensity**
- Average 0.06 actions per round means the AI takes ~31 tower actions per 512-round match
- DowngradeTower(4724) > BuildTower(4332) > UpgradeTower(3506) — lots of tower recycling
- This confirms the "build cheap, upgrade to Mortar, sell and rebuild elsewhere" pattern from code analysis

**7. Search Tree Under-Utilization**
- Average 4363 nodes out of 20K max — the 150ms budget is the binding constraint, not node count
- Max seen: 19990 (some rounds do hit the limit)
- Average 81.2 children per root expansion — large branching factor limits depth to ~2 levels

**8. Attack Mode**
- Triggered in ALL 200 matches at avg round 411.5
- This is a late-game phenomenon driven by the kill differential crossing threshold -3 (adjusted for round)
- The R450 coin drop correlates with attack mode investment in ant upgrades and super weapons
- Emergency storm: 814 triggers (avg 4.07 per match) at avg round 308.8 — defensive storms to avoid imminent HP loss
